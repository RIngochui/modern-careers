# Phase 8 — University & Career Paths — UI Design Contract

> Visual and interaction contract for university/career entry prompts, path traversal UI, degree selection, and role unlock feedback. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none |
| Font | Segoe UI, system-ui, monospace |

**Rationale:** Project uses vanilla CSS with inline styles in host.html and player.html. No framework. Phase 8 maintains visual consistency with the dark theme established in Phases 1-7. All new UI elements are added as inline `<style>` blocks and DOM elements injected/toggled by game.ts socket handlers.

## Interaction Contract

### Career Entry Flow (Player Screen)

1. Player lands on career entry tile (e.g., Tile 12 Finance Bro)
2. Server emits `careerEntryPrompt` with: `{ career, requirements, fee, meetsRequirements, canAfford }`
3. **If requirements met + can afford:**
   - `#career-choice` container shown with career name, requirement text, fee, Enter + Pass buttons
   - Player clicks "Enter [Career]" -> socket emits `career-enter` with `{ career }`
   - Player clicks "Pass" -> socket emits `career-pass`
   - Prompt hides after choice
4. **If requirements NOT met:**
   - `#career-choice` container shown with requirement text in #f87171, no buttons
   - Auto-hides after 2 seconds
   - Turn auto-advances (server-side)

### Streamer Entry Flow (Player Screen)

1. Player lands on Tile 38 (Streamer)
2. Server emits `streamerEntryPrompt` with: `{ attemptsRemaining, costPerAttempt }`
3. `#streamer-roll` container shown
4. Player clicks "Roll":
   - Socket emits `streamer-roll-attempt`
   - Server responds with `streamerRollResult` `{ roll, success, attemptsRemaining }`
   - Result text updates
   - If success: prompt closes, player enters path
   - If fail + attempts remaining: roll button stays enabled, attempt counter updates
   - If fail + no attempts: prompt closes, turn advances
5. Player clicks "Pass": socket emits `career-pass`, prompt closes

### Cop Wait Flow (Player Screen)

1. Player chooses to enter Cop (clicks "Enter Cop")
2. Server deducts $15,000, sets wait state
3. `#turn-indicator` shows "Cop training: skip 1 turn before entering"
4. Next turn: player's turn is skipped (server handles), indicator updates to "Training complete"
5. Following turn: player begins rolling 1d6 inside Cop path

### Path Traversal Flow (Player Screen)

1. Player is in a path (`inPath: true`)
2. On their turn: Roll Dice button enabled (existing), rolls 1d6 (server enforces 1d6 for path turns)
3. Server processes path tile landing, emits `pathTileEvent` with: `{ career, tileIndex, totalTiles, eventText, statChanges, specialEffect }`
4. `#active-tile-instruction` updates with path tile event (see Component Inventory #5)
5. Path progress indicator updates (Component Inventory #4)
6. Stat grid updates from `getFullState` broadcast (existing pattern)

### Degree Selection Flow (Player Screen)

1. Player reaches final tile of University path
2. Server emits `degreeSelectionPrompt` with: `{ availableDegrees, hasDegree }`
3. **If hasDegree is false:**
   - `#degree-choice` container shown with 7 degree buttons
   - Player clicks a degree -> socket emits `choose-degree` with `{ degree }`
   - Prompt hides
   - If Medical Degree chosen: server sets `isDoctor = true`, moves player to Hospital
4. **If hasDegree is true:**
   - No prompt shown; path exits normally

### Path Completion Flow (Player Screen)

1. Player reaches final tile of any career path
2. Server emits `pathComplete` with: `{ career, netStats, roleUnlock, experienceCardStub }`
3. `#active-tile-instruction` shows completion summary (heading, stat summary, role unlock text)
4. Path progress indicator hides
5. `#stat-career` updates to completed career name
6. Player returns to main board loop on next turn

### Special: Mid-Path Hospital Redirect

1. During any path tile event, if HP falls to 0 or below
2. Server triggers existing Hospital redirect (`movedToHospital` event)
3. `inPath` set to false, `currentPath` set to null (D-13 pattern)
4. Path progress indicator hides
5. Normal Hospital flow takes over (existing Phase 6 UI)
6. `#stat-career` updates to "None" (Unemployed)

### Host Screen Updates

- Turn history entries added via existing pattern: `turnHistory.unshift(...)` in `initHostGame`
- New entry formats:
  - `"[Player] entered [Career]"`
  - `"[Player] [Career] T[N]: [short event]"`
  - `"[Player] completed [Career]! [Role unlock if any]"`
  - `"[Player] chose [Degree] degree"`
  - `"[Player] is now a Doctor! Sent to Hospital."`
